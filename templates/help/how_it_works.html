How it Works

 pub2sum employs different technologies which are introduced here briefly.

Summarization
The Layout Parser library provides a pipeline of Optical Character Recognition (OCR) tools to filter out the text of a paper. This text is then split into chunks and fed to the OpenAI API for summarization. After one round the algorithm checks if the resulting summaries are still too long for GPT-3 to process in one request. If not, the text is split again into chunks and summarized. This process is repeated until only one chunk of text is left, which is used for a last request to obtain the final summary of the paper.

Fetching references
First, the pdf2doi library is used to obtain the DOI of the paper. This identifier is then used to fetch the relevant metadata of the scientific paper via API calls. The response is formatted into three references. The first reference in the text, the continuing reference, and the reference list entry.
